Admin Revocation
You manually revoke a licence from the admin dashboard — e.g. you've issued a refund, detected fraud, or there's a policy violation. This immediately blocks Pro access on the customer's next check-in.
● You (Admin) ● Worker / Dashboard ● D1 Database ● Stripe
Step-by-Step Flow
8 steps. This is a manual action — you're in control throughout. Click any step for detail.
1
● Admin
Open the admin dashboard
Navigate to https://api.modelcollectr.co.uk/admin with the X-Admin-Secret header set. The easiest way is the ModHeader browser extension.
Using ModHeader (Chrome/Firefox extension):
  • Add header: X-Admin-Secret
  • Value: MC_ADMIN_3F21F96F91C4DFC7BED4B0E4FC829A36
  • Enable the rule, then visit the URL
Or via curl: curl -H "X-Admin-Secret: MC_ADMIN_3F21F96F91C4DFC7BED4B0E4FC829A36" https://api.modelcollectr.co.uk/admin

The dashboard loads with all licences listed. The admin secret is cached in sessionStorage after you enter it once via the prompt — you won't be asked again until you close the tab.
2
● Admin
Find the licence you want to revoke
Use the status filter buttons (All / Active / Revoked / Suspended) to narrow the list, then identify the correct row by email or key.
Each row shows: licence key, email (fetched live from Stripe), status badge, creation date, last check-in date, total check-in count, and action buttons. The email is displayed live — it's always up-to-date from Stripe, never stored in D1.
3
● Admin
Click the "Revoke" button on that row
A confirmation dialog appears before anything is sent. You must confirm before the revocation is applied.
The browser shows: "Revoke this key? The customer will lose Pro access on their next check-in." — OK or Cancel. This prevents accidental revocations.
4
● Worker
Dashboard sends POST /admin/licenses/:key/revoke
The browser posts the revoke request to the Worker with the X-Admin-Secret header and a notes field explaining the reason.
POST /admin/licenses/MC-K8HN-4XRQ-BZTW-7YJP/revoke
X-Admin-Secret: MC_ADMIN_3F21F96F91C4DFC7BED4B0E4FC829A36
Content-Type: application/json

{"notes": "Manually revoked from dashboard"}
The Worker re-validates the admin secret on every action endpoint — it doesn't trust a session.
5
● D1 Database
Worker updates the licence status to 'revoked'
D1 is updated with the new status and the optional note you provided. The change is instant.
UPDATE licenses
SET status = 'revoked',
    notes = 'Manually revoked from dashboard'
WHERE key = 'MC-K8HN-4XRQ-BZTW-7YJP'
Notes are capped at 500 characters. The notes field is visible in the admin dashboard to remind you why a key was revoked.
6
● Admin
Dashboard shows "Done — reload to see changes" toast
A toast notification confirms the action succeeded. Reload the page to see the updated status badge.
The Worker returns {"ok": true} and the dashboard shows a brief toast. The row in the table still shows the old status until you reload — this is by design (reload to confirm).
7
● Stripe — Important!
Manually cancel the subscription in Stripe Dashboard
Revoking in D1 blocks Pro access but does NOT stop future billing. You must separately cancel the Stripe subscription to stop charges.
In Stripe Dashboard → Customers → find the customer → Subscriptions → Cancel subscription. Choose "Cancel immediately" for a refund scenario, or "at period end" if they've paid for a period they'll lose access to.

Why two steps? The admin dashboard is deliberately separate from Stripe billing. This gives you flexibility — e.g. you can revoke access while still letting a refund dispute resolve before cancelling in Stripe.
8
● Customer App
Customer's next check-in blocks Pro access
On the next app startup, /license/verify returns {valid: false, status: 'revoked'}. Pro features are locked immediately.
{"valid": false, "status": "revoked"}
The customer loses access at their very next app launch — not instantly. If they have the app open, it'll hit the API at the configured check-in interval (e.g. next startup). The app should show: "Your licence key is no longer valid. Contact support if you believe this is an error."
State After This Scenario
SystemState
D1 DatabaseStatus: revoked. Notes: reason you entered.
StripeSubscription still active (unless you cancelled it in step 7). Future billing will continue if not cancelled.
Customer AppPro locked on next check-in.
Always cancel in Stripe tooRevoking in the dashboard only blocks the key in D1. If you don't also cancel in Stripe, the customer will keep being charged every year. Do both steps every time.
Accidentally revoked?Use the Restore scenario — it's quick and the customer won't notice if you're fast enough (before their next app check-in).
