Restore a Revoked Licence
You accidentally revoked the wrong key, or a refund dispute was resolved in the customer's favour. The Restore action sets the key back to 'active' instantly. The customer's next check-in will succeed — no new key needed.
● You (Admin) ● Worker / Dashboard ● D1 Database ● Customer App
Step-by-Step Flow
5 steps. The quickest admin action available. Click any step for detail.
1
● Admin
Open the admin dashboard and find the revoked key
Filter by "Revoked" using the status buttons at the top of the table to narrow down the list quickly.
Navigate to https://api.modelcollectr.co.uk/admin with your admin header. Click the "Revoked" filter button to show only revoked licences. Find the correct row by email (fetched live from Stripe) or by the licence key.
2
● Admin
Click the "Restore" button on that row
Unlike Revoke, Restore has no confirmation dialog — it's reversible so it's safe to action immediately.
The Restore button appears green on revoked rows (in place of the Revoke button). Clicking it immediately sends the restore request — no browser confirm dialog.
3
● Worker
Dashboard sends POST /admin/licenses/:key/restore
The Worker receives the restore request, validates the admin secret, and updates D1.
POST /admin/licenses/MC-K8HN-4XRQ-BZTW-7YJP/restore
X-Admin-Secret: MC_ADMIN_3F21F96F91C4DFC7BED4B0E4FC829A36

Response: {"ok": true}
4
● D1 Database
Worker sets status = 'active' and clears the notes
The licence row is updated instantly. Notes are cleared so the revocation reason doesn't linger on a restored key.
UPDATE licenses
SET status = 'active',
    notes = NULL
WHERE key = 'MC-K8HN-4XRQ-BZTW-7YJP'
The change is immediate — D1 is updated before the response is returned to the browser.
5
● Customer App
Customer's next check-in succeeds — Pro restored
The very next time the app calls /license/verify, it gets {valid: true, status: 'active'} back. No new key, no customer action needed.
{"valid": true, "status": "active"}
If the customer had their app open during the revocation period, they might have seen a "key invalid" message. Tell them to restart the app — the next check-in will restore Pro access automatically.
