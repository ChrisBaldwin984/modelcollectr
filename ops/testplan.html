Purchase & Key Delivery
TC-01
Successful purchase — key delivered by email
Happy Path ▼ expand
Preconditions
Stripe in test mode. ModelCollectr website accessible. Admin dashboard accessible.
Steps
  1. Go to modelcollectr.co.uk → click "Buy Now"
  2. You should be redirected to Stripe's hosted checkout page
  3. Enter a real email address, card 4242 4242 4242 4242, any future date, any CVC
  4. Click Subscribe. You'll see "Your trial has started" on the Stripe page.
  5. Wait to be redirected to the success page (success.html?session_id=...)
  6. Confirm the licence key is displayed on screen with "Copy" button
  7. Confirm "We've emailed a copy to [your email]" message appears
  8. Check your email inbox for the licence email. Subject: "Your ModelCollectr Pro Licence Key"
  9. Confirm the email contains the same key shown on the success page
Expected Results
  • Success page displays key in MC-XXXX-XXXX-XXXX-XXXX format
  • Email arrives within 60 seconds with matching key
  • Email contains 4-step activation instructions
  • Admin dashboard shows new active row with email

Purchase ▼ expand

TC-04
Success page — key shows correctly while waiting for webhook
Purchase ▼ expand
Steps
  1. Complete a successful purchase (TC-01 steps)
  2. Observe the success page — it should show a spinner initially
  3. Wait for the key to appear (usually < 3 seconds)
  4. Note: the spinner indicates the page is polling /license/lookup
Expected Results

App Activation & Verification
TC-05
Valid key activates Pro features in the app
Happy Path ▼ expand
Preconditions
Complete TC-01 first. Have the licence key ready.
Steps
  1. Open ModelCollectr desktop app
  2. Go to Help → Enter Licence Key
  3. Paste the key received in TC-01

TC-06
Invalid key format is rejected immediately
Security ▼ expand
Steps
  1. Send a POST to https://api.modelcollectr.co.uk/license/verify with body {"key": "INVALID-KEY"}
  2. Also test with: empty string, very long string (1000 chars), SQL injection attempt '; DROP TABLE licenses; --, key with correct length but wrong prefix
Expected Results
  • All invalid formats return HTTP 200 with {"valid": false, "reason": "invalid_format"}
  • No database query is made (Worker rejects before DB lookup)
  • No 500 errors triggered

TC-10
Revoke a licence from the admin dashboard
Admin ▼ expand
Preconditions
Active licence from TC-01 available in admin dashboard.
Steps
  1. Open admin dashboard
  2. Find the active licence from TC-01
  3. Click "Revoke" button
  4. Confirm the browser dialog
  5. Reload the dashboard
  6. Now try to verify the key: POST to /license/verify with that key
Expected Results
  • Toast notification: "Done — reload to see changes"
  • After reload: status badge shows "revoked" (red)
  • Verify endpoint returns {"valid": false, "status": "revoked"}
TC-11
Restore a revoked licence
Admin ▼ expand
Preconditions
A revoked licence is available (e.g. from TC-10).
Steps
  1. Open admin dashboard, filter to "Revoked"
  2. Find the revoked licence, click "Restore"
  3. Reload the dashboard
  4. Verify the key again via POST to /license/verify
Expected Results
  • Toast: "Done — reload to see changes"
  • After reload: status badge shows "active" (green)
  • Notes field cleared
  • Verify endpoint returns {"valid": true, "status": "active"}

Billing Events (Stripe Test)
TC-13
Simulate payment failure — licence becomes suspended
Billing ▼ expand
Steps
  1. In Stripe Dashboard (test mode) → find the subscription from TC-01
  2. Click the subscription → Actions → "Simulate payment failure" (or use Stripe CLI)
  3. Stripe CLI: stripe trigger invoice.payment_failed --add invoice:subscription=sub_...
  4. Check admin dashboard (reload)
  5. Verify the key via /license/verify
Expected Results
  • Admin dashboard shows status: "suspended" (amber badge)
  • Verify returns {"valid": false, "status": "suspended"}
TC-14
Simulate payment recovery — licence returns to active
Billing ▼ expand
Preconditions
Licence is suspended (from TC-13).
Steps
  1. Stripe CLI: stripe trigger invoice.payment_succeeded --add invoice:subscription=sub_...
  2. Or in Stripe Dashboard: retry the failed invoice manually
  3. Check admin dashboard (reload)
  4. Verify key via /license/verify
Expected Results
  • Admin dashboard shows status: "active" (green badge)
  • Verify returns {"valid": true, "status": "active"}
TC-15
Subscription cancellation — licence revoked at period end
Billing ▼ expand
Steps
  1. In Stripe Dashboard → find the subscription → Cancel → Cancel immediately (for test)
  2. Stripe fires customer.subscription.deleted webhook
  3. Check admin dashboard (reload)
  4. Verify key via /license/verify
Expected Results
  • Admin dashboard shows status: "revoked" (red badge)
  • Notes column shows "Subscription cancelled by Stripe event"
  • Verify returns {"valid": false, "status": "revoked"}
Edge Cases & Security
TC-16
Webhook with invalid signature is rejected
Security ▼ expand
Steps
  1. Send a POST to /webhooks/stripe with a fake/invalid Stripe-Signature header
  2. e.g. curl -X POST https://api.modelcollectr.co.uk/webhooks/stripe -H "Stripe-Signature: t=1,v1=invalid" -d '{"type":"test"}'
Expected Results
  • HTTP 400 "Invalid signature"
  • No licence is created or modified
TC-17
Duplicate webhook does not issue a second key (idempotency)
Security ▼ expand
Steps
  1. Complete a purchase (TC-01) — note the key created and subscription ID
  2. Using Stripe Dashboard → Webhooks → find the customer.subscription.created event → Resend
  3. Check admin dashboard for duplicate rows
Expected Results
  • Only one row in admin dashboard for this subscription
  • No second key created
  • No second email sent
TC-18
CORS — only allowed origins can call the API
Edge Case ▼ expand
Steps
  1. From a browser console on a different site, try: fetch('https://api.modelcollectr.co.uk/checkout', {method:'POST'})
  2. Compare with calling from modelcollectr.co.uk
Expected Results
  • Request from unauthorised origin: CORS error in browser console, request blocked
  • Request from modelcollectr.co.uk: succeeds with correct CORS headers
