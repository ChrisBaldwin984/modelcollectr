Section 1

How the system works — the big picture

ModelCollectr Pro is a £19.99/year subscription sold on your website. When someone pays, the system automatically generates a unique licence key, emails it to them, and stores it in a database. The desktop app uses that key to unlock Pro features.

There are four main parts working together:

PartWhat it doesWhere it lives
Marketing siteWhere customers discover and buy the appmodelcollectr.co.uk
StripeHandles all payments, subscriptions, and renewalsdashboard.stripe.com
API WorkerThe brain — generates keys, talks to Stripe, sends emails, runs the admin dashboardapi.modelcollectr.co.uk
Desktop appChecks the key on every launch, enforces free/Pro limitsCustomer's Windows PC

You don't need to do anything manually for a normal purchase — the whole thing is automated. You only need to step in for refunds, abuse, or support queries.

Section 2

What happens when someone buys

Here's exactly what happens, step by step, from the customer's point of view and behind the scenes:

  1. Customer clicks "Get Pro" on your site. They're taken to a Stripe-hosted checkout page with your logo and brand colours.
  2. They enter their email and card details. There's a 14-day free trial — their card isn't charged immediately.
  3. They click Subscribe. Stripe processes it and redirects them back to your success page.
  4. The success page shows a spinner for 1–3 seconds while the system generates their key, then displays it.
  5. At the same time, an email from hello@modelcollectr.co.uk arrives in their inbox with the same key.
  6. They open the desktop app → Help → Enter Licence Key → paste the key → click Activate. Pro features unlock.
Key point: The key is tied to their email address and Stripe subscription. If their subscription lapses, the key stops working. If they resubscribe, they get a new key.

The 14-day free trial

Customers are not charged for 14 days. Their key is issued immediately and works straight away. After 14 days, Stripe charges their card. If the card fails and they don't fix it, the key gets suspended then eventually revoked.

What if the success page times out?

Occasionally the webhook from Stripe arrives slightly slowly. If the success page shows an error after 30 seconds, tell the customer: "Your key has been emailed to you — please check your inbox including spam." The email is sent regardless.

Section 3

The admin dashboard

How to access it

The dashboard is at https://api.modelcollectr.co.uk/admin but it requires a secret header to open. The easiest way is to use the ModHeader browser extension (Chrome or Firefox).

  1. Install ModHeader from the Chrome Web Store (search "ModHeader").
  2. Click the ModHeader icon in your toolbar.
  3. Add a header: Name = X-Admin-Secret, Value = your admin secret (see Section 7).
  4. Make sure the toggle is ON (green).
  5. Visit https://api.modelcollectr.co.uk/admin — the dashboard loads.
Remember: Turn ModHeader off when you're done, or it will send the secret header to every website you visit (harmless but unnecessary).

What you can see

The dashboard shows all licence keys with: email address, status (active/suspended/revoked), date created, last check-in date, and how many times the app has checked in.

At the top, four stats boxes show totals for active, suspended, and revoked keys.

You can filter by status using the buttons above the table.

Actions you can take

ButtonWhat it doesWhen to use it
RevokeImmediately marks the key as revoked. App locks within 7 days (or immediately on next launch).Refund issued. Chargeback. Abuse. Duplicate purchase.
RestoreSets a revoked key back to active. App unlocks on next launch.Revoked by mistake. Refund reversed. Dispute resolved in your favour.

Important: Revoking a key in the dashboard does NOT cancel their Stripe subscription — they could still be charged. Always cancel in Stripe as well if needed.
Section 4

Handling customer support

"I didn't receive my licence key email"

  1. Ask them to check spam/junk folder for an email from hello@modelcollectr.co.uk.
  2. If not there, open the admin dashboard and find their email in the table.
  3. Click Resend — they'll get the email again within a minute.
  4. If their email isn't in the table at all, check Stripe to confirm the payment went through.

"My key isn't working / says invalid"

  1. Check the admin dashboard — look up their email, check the status column.
  2. If suspended: their payment likely failed. Check Stripe — if they've updated their card and Stripe recovered payment, click Restore in the dashboard.
  3. If revoked: check why. If it was an error, click Restore.
  4. If active: ask them to copy the key from their email exactly (no extra spaces). The format is MC-XXXX-XXXX-XXXX-XXXX.

"I want a refund"

  1. Process the refund in Stripe Dashboard → Payments → find the charge → Refund.
  2. Then go to the admin dashboard → find their row → click Revoke.
  3. Also cancel their subscription in Stripe so they're not charged again.
Stripe's standard trial policy means customers can cancel within 14 days with no charge. After the trial, refunds are at your discretion.

"I've changed my email / PC"

The key itself doesn't change — it's not locked to a specific machine. They can enter the same key on a new PC. Just resend the email if they don't have the key saved.

"I cancelled but want to come back"

They go through checkout again, get a new key, and enter it in the app. The old key stays revoked. The new key works independently.

Section 5

Managing Stripe

Viewing subscriptions and customers

Stripe Dashboard → Customers — shows everyone who has subscribed. You can see their subscription status, payment history, and card details. You can also manually cancel, pause, or issue refunds from here.

Creating a promo code

  1. Stripe Dashboard → Product Catalogue → Coupons → + New coupon.
  2. Set the discount (e.g. 50% off, or £10 off).
  3. Then go to Promotion codes → + New code → link it to your coupon → give it a name (e.g. LAUNCH50).
  4. That code can be entered on the Stripe checkout page — the field appears automatically.
You can limit promo codes to a number of uses, an expiry date, or first-time customers only — all in Stripe without any code changes.

Stripe test vs live mode

The system is currently in test mode — no real money changes hands. The site is using test Stripe keys. When you're ready to go live, see Section 6.

The toggle in the top-right of Stripe Dashboard switches between Test and Live mode. Always check which mode you're in before making changes.
Section 6

Switching to live mode (when ready)

Do this only when you're confident testing is complete. Switching to live mode means real cards will be charged.
  1. In Stripe, switch to Live mode (top-right toggle).
  2. Go to Product Catalogue → create the same product and price: ModelCollectr Pro, £19.99/year.
  3. Go to Developers → Webhooks → Add endpoint: https://api.modelcollectr.co.uk/webhooks/stripe — select the same events as test mode.
  4. Copy your live Secret key (starts sk_live_...) and live Webhook signing secret (starts whsec_...).
  5. On your server/terminal, run these two commands to update the Worker secrets:
    Terminal commands
    cd ~/projects/modelcollectr-worker
    npx wrangler secret put STRIPE_SECRET_KEY
    npx wrangler secret put STRIPE_WEBHOOK_SECRET
  6. Update STRIPE_PRICE_ID in wrangler.toml to the new live price ID.
  7. Deploy: npx wrangler deploy
  8. Update Stripe branding in both Test and Live mode: Settings → Branding → upload logo, set colours.
