Before you start

This document covers all test scenarios for the ModelCollectr Pro licence system. The system is in Stripe test mode — no real money is charged. Use the test cards below. Report any failures to the developer with the test case ID and what you observed.

What you need: A Windows PC with the ModelCollectr desktop app installed. A web browser (Chrome recommended). Access to an email inbox. The admin dashboard URL and secret (provided separately).

Always use the test card numbers below — never enter a real card during testing.
Stripe Test Cards
4242 4242 4242 4242
✓ Successful payment. Any future expiry, any 3-digit CVC, any postcode.
4000 0000 0000 9995
✗ Always declines. Use to test payment failure scenarios.
4000 0025 0000 3155
⚠ Requires 3D Secure authentication. Tests the auth flow.
4000 0000 0000 0341
✗ Attaches OK but payment fails. Use for subscription payment failure.
Group 1 — Purchase & Key Delivery 4 tests
TC-01
Successful purchase — key displayed on success page
HIGH
Steps
  1. Go to modelcollectr.co.uk
  2. Click "Get Pro — £19.99/yr" button (nav or hero)
  3. On Stripe checkout: enter any email, card 4242 4242 4242 4242, any future expiry, any CVC
  4. Click Subscribe
  5. Observe the success page
Expected results
  • Redirected to modelcollectr.co.uk/success?session_id=...
  • Spinner shows "Activating your licence…"
  • Within 10 seconds: key in format MC-XXXX-XXXX-XXXX-XXXX appears
  • Green tick icon visible
  • "Copy Key" button present and works
  • Email address shown below key box
  • Activation steps visible
TC-02
Licence key email received
HIGH
Steps
  1. Complete TC-01 using a real email inbox you can access
  2. Check the inbox (and spam) for the email address used at checkout
  3. Open the email
Expected results
  • Email arrives within 60 seconds
  • From: ModelCollectr <hello@modelcollectr.co.uk>
  • Subject: "Your ModelCollectr Pro Licence Key"
  • Key displayed in amber/gold monospace text — matches the key on the success page
  • Activation steps listed correctly
  • Subscription details and cancel info present
  • Logo visible in email header
TC-03
Copy key button works
MEDIUM
Steps
  1. On the success page from TC-01
  2. Click "Copy Key" button
  3. Open Notepad (or any text app)
  4. Paste (Ctrl+V)
Expected results
  • Button changes to "✓ Copied!" with green background
  • Button reverts to "Copy Key" after ~2.5 seconds
  • Pasted text matches the key shown on screen exactly
  • Format is MC-XXXX-XXXX-XXXX-XXXX (19 characters total)
TC-04
Promo code applies discount at checkout
MEDIUM
Steps
  1. Create a test promo code in Stripe Dashboard first (ask operator)
  2. Click "Get Pro" on the site
  3. On Stripe checkout, click "Add promotion code"
  4. Enter the promo code
  5. Complete checkout with card 4242 4242 4242 4242
Expected results
  • Promo code accepted — price updates to show discount
  • Checkout completes successfully
  • Success page shows key as normal
  • Key email received as normal
Group 2 — Desktop App Activation & Feature Gating 5 tests
TC-05
Free tier enforces 30-item limit
HIGH
Steps
  1. Open ModelCollectr desktop app (no licence activated)
  2. Add items to any collection until you reach 30 total
  3. Try to add a 31st item
Expected results
  • First 30 items add without issue
  • On the 31st: a modal appears explaining the limit
  • Modal contains "Get Pro" button that opens the website
  • 31st item is NOT added to the collection
TC-06
Free tier blocks locked reports and features
HIGH
Steps
  1. With no licence active, open the Reports section
  2. Try clicking the Insurance report tab
  3. Try clicking Print Report
  4. Try clicking Export CSV
  5. Try adding a Custom Field to an item
Expected results
  • Each locked feature shows the Pro gate modal
  • Modal clearly states which feature requires Pro
  • "Get Pro" button opens the website in browser
  • None of the locked actions actually execute
TC-07
Licence key activates Pro features
HIGH
Steps
  1. Get the test key from TC-01
  2. In the desktop app: click Help menu → Enter Licence Key
  3. Paste the MC-XXXX-XXXX-XXXX-XXXX key
  4. Click Activate
  5. After activation, try all previously blocked features
Expected results
  • Activation shows success message
  • Insurance report tab now accessible
  • Print Report works
  • Export CSV works
  • Custom fields can be added
  • Item limit removed (can add more than 30)
TC-08
Invalid key is rejected
HIGH
Steps
  1. In the licence key dialog, enter a made-up key: MC-FAKE-WXYZ-TEST-0000
  2. Click Activate
  3. Also try entering a totally random string: hello world
  4. Click Activate
Expected results
  • Both attempts are rejected with a clear error message
  • No Pro features unlock
  • App remains in free tier
TC-09
Pro persists after app restart
HIGH
Steps
  1. With a valid key activated (TC-07 complete)
  2. Fully close the desktop app
  3. Re-open it
  4. Check Pro features are still active
Expected results
  • App opens with Pro active — no need to re-enter key
  • All Pro features accessible immediately
  • No login or re-activation prompt
Group 3 — Admin Dashboard 4 tests
TC-10
Admin dashboard loads and shows customer row
HIGH
Steps
  1. Install ModHeader browser extension
  2. Add header: X-Admin-Secret = [secret provided by operator]
  3. Visit https://api.modelcollectr.co.uk/admin
  4. Find the row for the email used in TC-01
Expected results
  • Dashboard loads (not a 401 error)
  • Stats bar shows correct totals
  • Customer row visible with correct email
  • Status shows "active" (green badge)
  • Check-in count greater than 0 (app has checked in)
  • Revoke and Resend buttons visible
TC-11
Resend email from admin dashboard
HIGH
Steps
  1. In the admin dashboard, find the test customer row
  2. Click Resend
  3. Enter the admin secret when prompted (first time only)
  4. Check the email inbox for the email address in that row
Expected results
  • "Email resent!" toast notification appears
  • Second licence email arrives in inbox within 60 seconds
  • Key in the email is identical to the original key
TC-12
Revoke key — app loses Pro access
HIGH
Steps
  1. In admin dashboard, click Revoke on the test customer row
  2. Confirm the confirmation dialog
  3. Reload the admin page and check the row status
  4. In the desktop app: close and re-open it
  5. Try accessing a Pro feature (e.g. Insurance report)
Expected results
  • "Done — reload to see changes" toast appears
  • After reload: row shows "revoked" status (red badge)
  • After app restart: Pro features are blocked
  • App behaves as free tier
TC-13
Restore key — app regains Pro access
HIGH
Steps
  1. With key revoked from TC-12, go back to admin dashboard
  2. Find the row (status: revoked), click Restore
  3. Reload admin page and check status
  4. Close and re-open the desktop app
  5. Try a Pro feature
Expected results
  • "Done — reload to see changes" toast appears
  • After reload: row shows "active" status (green badge)
  • After app restart: Pro features work again
  • No need to re-enter the key
Group 4 — Edge Cases & Error Handling 3 tests
TC-14
Checkout abandonment — no key issued
MEDIUM
Steps
  1. Click "Get Pro" on the site
  2. When Stripe checkout loads, close the tab or click Back
  3. Check the admin dashboard
Expected results
  • No new key appears in the admin dashboard
  • No spurious email sent
  • Site continues working normally
TC-15
Declined card — checkout fails gracefully
MEDIUM
Steps
  1. Click "Get Pro" on the site
  2. Use the declining card: 4000 0000 0000 9995
  3. Any email, any future expiry, any CVC
  4. Click Subscribe
Expected results
  • Stripe checkout shows a clear error message (not a crash)
  • Customer can try again with a different card
  • No key issued, no email sent
  • Admin dashboard unchanged
TC-16
Admin dashboard rejects wrong secret
MEDIUM
Steps
  1. In ModHeader, change the X-Admin-Secret value to WRONG_SECRET
  2. Visit https://api.modelcollectr.co.uk/admin
  3. Also try performing a revoke action with the wrong secret
Expected results
  • Dashboard page returns "Unauthorised" (not the dashboard)
  • Revoke/restore/resend actions with wrong secret return an error
  • No data is changed
