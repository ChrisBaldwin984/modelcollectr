System Overview

ModelCollectr Pro is a desktop app that requires an annual subscription. The subscription system has four parts that talk to each other:

Cloudflare Worker (API)
The brain. Handles checkout, licence verification, email, admin dashboard. Lives at api.modelcollectr.co.uk. Runs 24/7 for free on Cloudflare's edge.
Stripe
Handles payments, subscriptions, and billing. Customers never touch your server for payment — Stripe handles it. Sends payment events back to the Worker via webhooks.
D1 Database (Cloudflare)
Stores licence keys, Stripe IDs, status, and check-in stats. Does NOT store email addresses (GDPR — emails stay in Stripe and are fetched on demand).
Resend
Transactional email service. Sends the licence key email from hello@modelcollectr.co.uk. Used for initial delivery and admin-triggered resends.
GDPR note: Email addresses are stored only in Stripe, never in your D1 database. The admin dashboard fetches them live from Stripe when you load it. This means you are not a "data processor" for personal data — Stripe is, and they have their own GDPR compliance. You hold only anonymous licence keys and subscription IDs.
2
What Happens When Someone Buys

From the moment they click Buy Now to unlocking Pro in the app — all automated, no action from you needed.

1
Customer clicks Buy Now on modelcollectr.co.uk
JavaScript calls the Worker API to create a Stripe Checkout Session. Customer is redirected to Stripe's hosted payment page.
2
Customer enters card details on Stripe's page
14-day free trial — no charge today. Stripe handles 3D Secure automatically. Card details go directly to Stripe, never to your server.
3
Stripe fires a webhook to the Worker
A customer.subscription.created event is sent. The Worker verifies the signature, then processes it.
4
Worker generates a licence key and stores it
A cryptographically random key in MC-XXXX-XXXX-XXXX-XXXX format is generated and written to D1. Only the key and Stripe IDs are stored — no email.
5
Licence key email is sent via Resend
Email goes to the address the customer used on Stripe's checkout page. Contains the key, activation steps, and support contact.
6
Customer lands on the success page
Stripe redirects to modelcollectr.co.uk/success?session_id=.... The page polls the Worker until the key is ready, then displays it on screen.
7
Customer activates in the desktop app
Help → Enter Licence Key → paste key → Activate. App calls the Worker to verify the key and unlocks Pro features.

For the full technical breakdown, see the Successful Purchase scenario.

Admin Dashboard

URL: https://api.modelcollectr.co.uk/admin — requires the X-Admin-Secret header.

How to access: Install the ModHeader extension (Chrome or Firefox). Add a header rule: Name = X-Admin-Secret, Value = MC_ADMIN_3F21F96F91C4DFC7BED4B0E4FC829A36. Enable the rule, then visit the URL. The dashboard will prompt for the secret the first time; it's cached in the tab session after that.

The dashboard shows:

●
Stats bar at the top
Total keys, Active count, Revoked count, Suspended count.
●
Filter buttons
All / Active / Revoked / Suspended — click to filter the table.
●
Licence table
Key, Email (live from Stripe), Status badge, Created date, Last check-in date, Check-in count, Action buttons.

Action buttons on each row:

Revoke
Sets status = revoked. Shows confirmation dialog first. Customer loses Pro on next check-in. Remember to also cancel in Stripe to stop billing.
Restore
Sets status = active on a revoked key. Customer regains Pro on next check-in. No new key needed. Check Stripe subscription is still active.
Resend
Fetches customer email live from Stripe and sends a fresh copy of their key. Use when customer says they lost the email.
Important: The Revoke button only blocks the key in your database. It does not cancel the Stripe subscription. Always cancel in Stripe separately to stop future billing charges.
4
Support Playbook

Common support requests and exactly what to do for each one.

Lost key"I can't find my licence key email"
  1. Open the admin dashboard
  2. Find their row (match their email address to the Email column)
  3. Click Resend
  4. Reply: "I've resent your key to [email]. Check spam if it doesn't arrive in a few minutes."
Invalid key"I entered my key but it says invalid"
Check in admin dashboard: find their row and check the status badge.
  • Active → They may have typed it wrong. Ask them to copy-paste from the email. Check they're using the current key (not an old one from re-subscription).
  • Suspended → Payment failed. Tell them to update their card. Stripe should have emailed them.
  • Revoked → Was it revoked by you? If in error, click Restore. If they cancelled, they need to resubscribe.
Refund"I want a refund"
  1. In Stripe Dashboard → Payments, find their payment and issue a refund
  2. In Stripe Dashboard → Subscriptions, cancel their subscription
  3. In your admin dashboard, click Revoke on their key
Stripe handles the money. You handle blocking access.
New PC"I got a new computer — does my key still work?"
Yes — the key is not machine-locked. They just need to install ModelCollectr on the new PC, go to Help → Enter Licence Key, and paste the same key. If they don't have the key, use the Resend action in the admin dashboard.
Cancelled"I cancelled but want to come back"
Direct them to modelcollectr.co.uk to purchase again. They'll go through the standard checkout and receive a new licence key. Their old key will not be reactivated — they get a fresh one.
App locked"My app suddenly says my licence is invalid and I'm paying"
  1. Check admin dashboard — is status suspended or revoked?
  2. Check Stripe — is the subscription past_due or cancelled?
  3. If suspended (payment failed): tell them to update their card via Stripe's link in the failure email they received
  4. If active in both places and still invalid: ask them to restart the app — sometimes the cache is stale
5
Stripe Management

Test mode vs Live mode: Stripe has two environments. Test mode uses fake money and test cards. Live mode uses real money. Always make sure you're in the right mode for what you're doing. The toggle is top-left in the Stripe Dashboard.

Viewing customers and subscriptions: Stripe Dashboard → Customers. Search by email to find a specific person. Click through to see their subscriptions, payment history, and invoices.

Issuing a refund: Stripe Dashboard → Payments → find the charge → Refund. Stripe processes it automatically to the customer's card. Note: you still need to revoke the key in your admin dashboard.

Cancelling a subscription: Stripe Dashboard → Subscriptions → find the subscription → Cancel. Choose "at period end" unless you're also issuing a full refund (then choose "immediately").

Promotion codes: Stripe Dashboard → Products → Coupons → Create coupon. Then Products → Promotion codes → Create code. Codes are automatically accepted at checkout (the "Allow promotion codes" setting is enabled).

Switching to Live mode: When ready to go live, you'll need to create a new Price in live mode, update the STRIPE_PRICE_ID environment variable in Cloudflare, create a new webhook endpoint in live mode Stripe, and update the STRIPE_WEBHOOK_SECRET and STRIPE_SECRET_KEY secrets. See the Going Live checklist.
